# KATA-DRAW モジュール分離案

## 1. 方針

- **責務ごとに分離**: ステージ、操作、表示、オーディオ、設定を独立したモジュールに
- **改変しやすさ**: 変更頻度の高い部分を明確に切り出し
- **段階的実施**: 体験版前は必須・推奨を優先し、製品版で拡張

---

## 2. モジュール構成案

```
┌─────────────────────────────────────────────────────────────────┐
│  game.gd（オーケストレーター）                                      │

└─────────────────────────────────────────────────────────────────┘
         │              │              │              │
         ▼              ▼              ▼              ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ StageModule  │ │ InputModule  │ │  UIModule    │ │ AudioModule  │

└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘

                ▼
         ┌──────────────┐
         │ Config/Data  │
         │ 設定・定数   │
         └──────────────┘
```

## 3. 各モジュールの詳細

### 3.1 Config / Data（設定・定数）

**形態**: リソース + 定数ファイル

| ファイル | 内容 |
|----------|------|
| `resources/stage_config.gd` または `resources/stages.json` | ステージ定義（STAGES 相当） |
| `resources/game_config.gd` | ヒントタイミング、パーティクル設定、体験版/製品版のステージ数など |
| `resources/theme_colors.gd` | 色定義（BG_COLOR, LINE_COLOR など） |
| `resources/ui_constants.gd` | POINT_RADIUS, HOVER_DISTANCE, BB_ANCHOR_SIZE など |

**役割**:
- ステージの追加・並び替え・バランス調整を一箇所で実施
- 体験版と製品版の切り替え（例: `max_stage_index`）
- 色・サイズの変更をコード全体から分離

---

### 3.2 StageModule（ステージ）

**形態**: `scripts/stage_manager.gd`（Autoload または game の子ノード）

| 担当 | 現 game.gd の該当 |
|------|-------------------|
| ステージ定義の読み込み | STAGES 定数 |
| 点の生成 | `_generate_polygon_group`, `_generate_two_circles`, `_generate_star_shape` |
| メトリクス計算 | `_calculate_metrics`, `_calc_group_metrics`, `_calculate_star_metrics` |
| クリア判定 | `_is_clear`, `_is_group_clear`, `_is_locked` |
| 星の幾何 | `_star_distance_at_angle`, `_interp_distance` |
| 点の精度表示用 | `_point_accuracy_alpha` |

**保持する状態**:
- `point_positions`, `current_stage`, `stage_type`
- `current_centroid`, `current_avg_radius`, `current_circularity`, `current_smoothness` など
- `group_split`, `group1_cleared`, `group2_cleared`（two_circles 用）
- `star_rotation`, `star_outer_r`, `star_inner_r`（star 用）

**公開 API 例**:
- `start_stage(idx: int) -> void`
- `get_point_positions() -> Array[Vector2]`
- `update_point_position(idx: int, pos: Vector2) -> void`
- `calculate_metrics() -> void`
- `is_clear() -> bool`
- `get_metrics() -> Dictionary`（HUD 表示用）

---

### 3.3 InputModule（操作）

**形態**: `scripts/input_handler.gd`（RefCounted、game が保持）

| 担当 | 現状（input_handler.gd） |
|------|---------------------------|
| マウス | `handle_mouse_motion`, `handle_mouse_press`, `handle_mouse_release` |
| ドラッグ・選択 | `_begin_drag`, ホバー＝選択（矩形選択は削除済み） |
| バウンディングボックス | `get_bb_rect`, `_hit_bb_anchor`, `_begin_bb_drag`, `_handle_bb_motion`, `_end_bb_drag` |
| 変形操作 | `rotate_selected`, `scale_selected`（入力未接続） |
| ゲームパッド | `_handle_pad_button`, `_cycle_pad_point`, `_nudge_pad`, `_process_pad` |
| ユーティリティ | `_find_point_at`, `_clamp_points_to_viewport`, `_get_selection_center` |

**保持する状態**:
- `drag_offsets`, `last_mouse_pos`（game が持つ: `selected_indices`, `is_dragging`, `hovered_index`）
- `bb_dragging`, `bb_anchor_idx`, `bb_origin`, `bb_start_positions` など
- `pad_cursor`, `pad_grabbing`, `pad_cursor_initialized`, `_last_input_method`

**公開 API 例**:
- `process_input(event: InputEvent) -> void`
- `process_pad(delta: float) -> void`
- `get_selected_indices() -> Array[int]`
- `get_hovered_index() -> int`
- `is_dragging() -> bool`
- `is_rect_selecting() -> bool`

**注意**: InputModule は StageModule の `point_positions` を更新する必要があるため、StageModule への参照を持つか、game が仲介する。

---

### 3.4 UIModule（表示）

**形態**: `scripts/ui/ui_renderer.gd` または CanvasLayer の子ノード

| 担当 | 現 game.gd の該当 |
|------|-------------------|
| ロゴ | `_draw_logo` |
| タイトル | `_draw_title` |
| メニュー | `_draw_menu` |
| コンフィグ | `_draw_config` |
| ルール | `_draw_rules` |
| ゲーム画面 | `_draw_game`, `_draw_guide_info`, `_draw_guide_countdown` |
| お手本・ヒント | `_draw_guide_shape`, `_draw_hint_shape` |
| HUD | `_draw_ui_panel`, `_draw_hud` |
| クリア・リザルト | `_draw_clear_overlay`, `_draw_results` |
| ポーズ | `_draw_pause_overlay` |
| パーティクル | `_draw_particles`, `_update_particles`, `_spawn_particles` |
| ユーティリティ | `_draw_string_fit`, `_draw_ring`, `_draw_star_outline`, `_draw_bounding_box` |
| クロスフェード | `_crossfade_alpha` |

**保持する状態**:
- `font`, `logo_texture`, `title_logo_texture`
- `particles`, `particle_spawn_time`
- メニュー系のインデックス（menu_index, config_index など）は game が持つか、UI が持つか要検討

**公開 API 例**:
- `draw(state: String, context: Dictionary) -> void`（state に応じて描画を振り分け）
- `spawn_particles(center: Vector2) -> void`
- `update_particles(delta: float) -> void`

**注意**: 描画は `_draw()` 内で行う必要があるため、UIModule が CanvasItem を継承するか、game が UIModule の draw メソッドを呼ぶ形になる。

---

### 3.5 AudioModule（音声）

**形態**: `scripts/audio/audio_manager.gd`（Autoload または game の子ノード）

| 担当 | 現 game.gd の該当 |
|------|-------------------|
| BGM | `_setup_audio`, `_play_bgm`, `_stop_bgm` |
| SE | `_play_sfx` |
| リソース読み込み | `_load_audio` |

**保持する状態**:
- `bgm_title`, `bgm_game`, `bgm_result`
- `sfx_count`, `sfx_clear`

**公開 API 例**:
- `play_bgm(type: String)`  # "title", "game", "result"
- `stop_bgm(type: String)`
- `play_sfx(type: String)`    # "count", "clear"

---

### 3.6 game.gd（オーケストレーター）

**残す役割**:
- ゲーム状態（`game_state`）の管理と遷移
- `_input()` でイベントを受け取り、状態に応じて各モジュールへ振り分け
- `_process()` でタイマー・ヒント・パーティクル更新などを調整
- `_draw()` で UIModule の描画を呼び出し
- メニュー・コンフィグ・ポーズの入力処理（`_input_menu`, `_input_config`, `_input_pause`）
- コンフィグ適用（`_apply_config_selection`, `_center_window`）

**削減される行数**: 約2000行以上を各モジュールへ移行

## 5. 依存関係とデータの流れ

```
InputModule ──(point_positions 更新)──► StageModule
                    │
                    │ selected_indices, hovered_index
                    ▼
game.gd ◄───────────────────────────── StageModule (metrics, is_clear)
    │
    ├──► AudioModule (BGM/SE 制御)
    │
    └──► UIModule (描画に state, metrics, point_positions などを渡す)
```

- **game.gd** が StageModule と InputModule を仲介
- InputModule が点を動かす → StageModule の `point_positions` を更新
- game.gd が StageModule から metrics を取得し、UIModule に渡して描画

## 6. 実施の優先度

| 段階 | モジュール | 理由 |
|------|------------|------|
| **1（必須）** | Config: ステージ定義 | 体験版のステージ切り替え、バランス調整に必須 |
| **2（推奨）** | Config: ゲーム設定・色・UI定数 | ヒントタイミング、体験版/製品版切り替え |
| **3（推奨）** | StageModule | ステージ追加・仕様変更の中心 |
| **4** | InputModule | 操作性改善時に変更が集中 |
| **5** | UIModule | 操作ガイド・お手本表示の見直し時に有効 |
| **6** | AudioModule | BGM 差し替え時に整理しやすい |

## 7. 検討事項

### 7.1 ノード構成

- **Autoload**: StageManager, AudioManager を Autoload にすると、どこからでも参照可能。ただし game 以外から参照する必要がなければ子ノードでも可。
- **子ノード**: InputHandler, UIRenderer は game の子として配置し、`get_parent()` で game を参照する形も可能。

### 7.2 描画の扱い

- Godot の `_draw()` は CanvasItem のメソッド。UIModule を別ノードにする場合:
  - **案A**: CanvasLayer を子に持ち、その子に CustomDraw するノードを置く
  - **案B**: game が `_draw()` 内で UIModule の static メソッドや、渡した context を元に描画する関数を呼ぶ
  - **案C**: 各画面（タイトル、メニュー、プレイなど）を別シーン化し、game がシーンを切り替える（大規模）

体験版前は **案B**（game が UIModule の関数を呼ぶ）が移行コストが低い。

### 7.3 メニュー・コンフィグ・ポーズの入力

- これらは「操作」というより「UI の選択操作」。InputModule に含めるか、game に残すか。
- **案**: メニュー系は game に残す。プレイ中の点操作のみ InputModule に分離。段階的に InputModule へ移行も可。
