# 実現率とクリア判定の仕様

## 用語

| 用語 | 説明 |
|-----|------|
| **clear_pct** | ステージ設定の目標値（例: さかな=98） |
| **clear_threshold** | 100 - clear_pct（例: 98→2） |
| **current_circularity** | 生の円形度 0〜100（高いほど良い） |
| **current_circularity_error** | 弧誤差（低いほど良い）= 100 - current_circularity |
| **display_rate_min_pct** | ステージごとの実現率表示の下限（デフォルト50、例: さかな75） |
| **実現率（表示値）** | min_pct以下→0、min_pct〜目標の範囲で 0〜100 にマッピング |

## 実現率の計算式

```
goal_pct = clear_pct (= 100 - clear_threshold)
min_pct  = display_rate_min_pct  # ステージごとに指定、未指定時50
実現率   = (current_circularity - min_pct) / (goal_pct - min_pct) * 100  （0〜100でクランプ）
```

- current_circularity ≤ min_pct → 実現率 0
- current_circularity ≥ goal_pct → 実現率 100

## 問題点（修正前）

1. **クリア判定**: `current_circularity_error < clear_threshold`  
   → つまり `current_circularity > clear_pct` でクリア
2. **実現率100**: `current_circularity >= clear_pct` で100表示
3. **不整合**: `current = 98.0` のとき実現率は100だが、`>` のためクリアしない
4. **クリアオーバーレイ**: 生の `current_circularity`（98.1など）を表示しており、実現率100と一致しない

## 修正方針

- クリア条件を **実現率 ≥ 100** に統一 = `current_circularity >= clear_pct`
- クリアオーバーレイには **実現率** を表示（クリア時は100になる）
