# KATA-DRAW コントローラ操作の実装

コントローラ（ゲームパッド）での操作がどのように実装されているかを整理する。

---

## 1. 実装の分担

| 担当 | ファイル | 対象 |
|------|----------|------|
| **game.gd** | `_input()` および各 `_input_*()` | 画面遷移・メニュー・設定・ポーズ |
| **InputHandler** | `handle_pad_button()`, `process_pad()` | プレイ中の操作 |

---

## 2. game.gd でのコントローラ処理

### 2.1 共通の判定

| 変数 | 用途 | 対象ボタン |
|------|------|------------|
| `is_confirm_pad` | 決定操作 | JOY_BUTTON_A |
| `is_pause_key` | ポーズの開閉 | JOY_BUTTON_START |

### 2.2 画面ごとの処理

| 画面 | ボタン | 動作 |
|------|--------|------|
| logo | A | タイトルへ |
| title | A | メニューへ |
| menu | A | 選択項目を実行 |
| | DPAD_UP/DOWN | 項目選択 |
| | B | タイトルへ |
| config | A | 選択項目を実行 |
| | DPAD_UP/DOWN | 項目選択 |
| | B | 戻る |
| rules | Start | ゲーム開始 |
| guide_info | A | カウントダウンへ |
| guide_countdown | （無効） | |
| playing | 下記 InputHandler へ委譲 | |
| | START | ポーズ表示 |
| cleared | A | 次へ |
| results | A | タイトルへ |
| pause | A | 選択項目を実行 |
| | DPAD_UP/DOWN | 項目選択 |
| | START | ポーズ解除 |
| pause_confirm | DPAD_UP/DOWN | Yes/No 選択 |
| | START | キャンセル |

### 2.3 呼び出しフロー

```
_input(event)
  ├─ game_state に応じて早期 return
  ├─ pause_active 時 → _input_pause()
  ├─ playing 時
  │   ├─ is_pause_key → ポーズ表示
  │   ├─ preferred_input_method == "mouse" なら Joypad/Mouse を input_handler に渡さない
  │   ├─ allow_pad 時のみ InputEventJoypadButton → input_handler.handle_pad_button(btn)
  │   └─ allow_mouse 時のみ InputEventMouse* → input_handler.handle_mouse_*
  └─ （マウス・ホイール等は別途）

_process(delta)
  └─ _process_pad(delta)
       ├─ preferred_input_method == "mouse" かつ playing なら return（パッド処理スキップ）
       └─ input_handler.process_pad(delta)
```

## 3. InputHandler でのコントローラ処理（プレイ中のみ）

### 3.1 ボタン入力（`handle_pad_button`）

**前提**: コントローラ操作時は常に 1 点以上を選択した状態で動作する。

| ボタン | 動作 |
|--------|------|
| **LB / RB** | 前 / 次のポイントへジャンプ（`_cycle_pad_point`） |
| **DPAD** | 指定方向にある隣接ポイントへ切り替え（A または右スティック選択中はポイント移動） |
| **A + DPAD** | 現在選択中のポイントを連続移動。右スティックで選択中も代替可能 |

### 3.2 アナログ入力（`process_pad`）

| 入力 | 用途 | 備考 |
|------|------|------|
| **左スティック** | 選択ポイント切り替え | 現在ポイントから倒した方向に直線を引き、その直線に最も近いポイントへ切り替え（連結は無視、相対距離のみで判定） |
| **A + 左スティック** | ポイント移動 | 選択中のポイントを傾きに応じて移動 |
| **右スティック** | ポイント選択・つかみ | 倒した方角に直線を引き、その直線に最も近いポイントを選択（隣接関係は無視、位置のみで判定）。倒し続ける間はそのポイントをつかんだ状態を維持。スティックを元に戻すとリリース。わずかな傾きでは反応しない（揺り戻し誤動作防止） |

### 3.3 定数（input_handler.gd）

| 定数 | 値 | 用途 |
|------|-----|------|
| PAD_CURSOR_SPEED | 600.0 | 左スティックのカーソル移動速度 |
| PAD_RIGHT_STICK_SPEED | 400.0 | A+左スティックのポイント移動速度 |
| PAD_A_DPAD_SPEED | 266.67 | A+十字キー連続移動速度（スティック全倒しの約2/3） |
| PAD_RIGHT_STICK_DEADZONE | 0.5 | 右スティックのデッドゾーン（揺り戻し・誤検知防止） |

### 3.4 状態変数

| 変数 | 用途 |
|------|------|
| pad_cursor | 画面上のカーソル位置 |
| pad_cursor_initialized | 初回のみ画面中央で初期化 |
| pad_grabbing | ポイントをつかんでいるか |
| _right_stick_was_active | 前フレームで右スティックがデッドゾーン超えていたか（ホールド中は再選択しない） |
| _last_input_method | 直前の入力手段（"mouse" / "pad"）。マウス入力を検知した際にコントローラの掴み状態をリリースするため |

### 3.5 パッドカーソルの表示

`ui_renderer.gd` の `_draw_game()` 内で、`Input.get_connected_joypads().size() > 0` のときのみパッドカーソル（十字）を描画。

---

## 4. 処理の流れ（プレイ中）

```
_process(delta)
  └─ _process_pad(delta)
       └─ input_handler.process_pad(delta)
            ├─ game_state != "playing" なら return
            ├─ 右スティック → 倒した方角の直線に最も近いポイントを選択・つかみ（ホールド中は維持、離すとリリース）
            ├─ 左スティック → A/右スティック選択中はポイント移動、それ以外は選択ポイント切り替え
            └─ A/右スティック選択 + DPAD → ポイント連続移動

_input(event)
  └─ （playing かつ JoypadButton）
       └─ input_handler.handle_pad_button(btn)
            ├─ LB/RB → ポイント切り替え
            └─ DPAD → A/右スティック選択中でなければ隣接ポイントへ切り替え
```

---

## 5. 補足

- **ジョイパッド ID**: 常に `0`（`Input.get_joy_axis(0, ...)` 等）
- **デッドゾーン**: 左スティック 0.15、右スティック 0.5（揺り戻し・誤検知防止）
- **入力ロック**: デモ画面（rules）で「次へ」をコントローラ/キーボードで選ぶと、プレイ中はパッドのみ有効。マウスで選ぶとマウスのみ有効（`preferred_input_method`）。詳細は `Docs/特殊操作一覧.md`
- **マウスとの併用**: 入力ロック未設定時はマウスとパッドの両方が有効
- **入力切り替え**: マウス入力を検知した瞬間に、コントローラで掴んでいた状態をリリースする
