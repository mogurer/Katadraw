# KatadrawShapeEditor（Shape Grid Editor）操作仕様

Katadraw で使う**目標図形**（閉じたポリライン＋任意の円弧エッジ）を、**主にマウス操作**で編集し、`stage_manager.gd` 等へ貼り付け可能な **GDScript 断片**として出力するためのエディタです。

- 実装: `KatadrawShapeEditor/scripts/shape_grid_editor.gd`
- エントリシーン: `KatadrawShapeEditor/scenes/shape_grid_editor.tscn`（プロジェクトの開き方によって `res://` からの相対パスが変わる場合あり。スタンドアロンで `KatadrawShapeEditor` をプロジェクトルートに開く場合は `res://scripts/shape_grid_editor.gd` を参照）

---

## 1. 目的と前提

| 項目 | 内容 |
|------|------|
| 目的 | 多角形の**頂点列**と、円弧になる**辺**の制御点を編集し、Katadraw の `get_*_polygon_vertices()` / `get_*_arc_controls()` 形式に近い出力を得る |
| 実行環境 | Godot 4.x（開発時は 4.6 想定）。シーンを選択して **F6**（現在のシーンを実行）で起動 |
| 操作領域 | ウィンドウ左 **約 75%** がキャンバス。右 **約 25%** はスクリプト用 `TextEdit`（実行時のみ生成） |

---

## 2. 画面レイアウト

| 領域 | 比率 | 内容 |
|------|------|------|
| 左 | `LEFT_RATIO = 0.75` | グリッド・図形・頂点番号。マウス編集は**この左領域のみ**有効（右パネル上のクリックは無視） |
| 右 | 残り | ダーク背景＋テキストエリア。出力された GDScript を表示・手編集可能 |
| 中央境界付近 | — | **▶**（セーブ）**◀**（ロード）ボタン。左パネルと右パネルの境界に重なる位置 |

---

## 3. 座標・データモデル（内部）

### 3.1 グリッド

| 定数 | 値 | 意味 |
|------|-----|------|
| `GRID_CELL_SIZE` | 24.0 | 1 マスあたりの画面ピクセル |
| `GRID_SIZE` | 16 | グリッドの「半径」。論理座標はおおよそ **-16 ～ 16**（描画は左パネル幅でクリップ） |
| 頂点座標 | 整数 `Vector2` | **グリッド座標**（マス目のインデックス）。初期形状は三角形 3 点 |

画面上は、エディタ左エリアの中心を原点に、**X 右・Y 上**が正になるよう変換（`shape_grid_editor.gd` の `_grid_to_screen` / `_screen_to_grid`）。

### 3.2 ポリゴンとエッジ

- **頂点** `_points`: 反時計回りなど一筆の閉じた輪郭の順。
- **エッジ** `_edges[i]`: 頂点 `i` → `(i+1) % n` の辺。
  - `{"type": "line"}` … 直線
  - `{"type": "arc", "arc_control": Vector2}` … 円弧。`arc_control` はグリッド座標の**第 3 点**（弦の両端と円弧上の点で円を定義。描画は「第 3 点を通らない側の弧」というルールで `_sample_arc_3points` によりサンプル）

---

## 4. マウス操作（左パネルのみ）

距離の閾値（抜粋）:

| 定数 | 値 | 用途 |
|------|-----|------|
| `POINT_RADIUS` | 8.0 | 頂点の描画半径 |
| `HOVER_DISTANCE` | 28.0 | **頂点ヒット**判定（クリック位置から最も近い頂点） |
| `EDGE_ADD_DISTANCE` | 80.0 | **辺ヒット**（直線は線分、円弧は弧の折れ線近似に対する距離） |
| `RIGHT_CLICK_DRAG_THRESHOLD` | 5.0 | 右ボタンで **ドラッグとみなす**までの移動量（ピクセル） |

### 4.1 左ボタン

| 操作 | 条件 | 結果 |
|------|------|------|
| **左クリック** | カーソルが**頂点付近**（上記ヒット） | その頂点を掴んで**左ドラッグ**開始 |
| **左クリック** | 頂点ではなく、**辺の近く** | その辺に新規頂点を挿入。**直線エッジ**として分割（挿入点の両側は `line`） |
| **左ドラッグ** | 頂点上で押下中 | 頂点を移動。終了時に弧まわりの `arc_control` を更新（下記 4.3） |

辺上追加時、クリック位置はグリッドにスナップ（円弧辺上の場合は弧上の最近点を取ってからスナップ）。

### 4.2 右ボタン

| 操作 | 条件 | 結果 |
|------|------|------|
| **右クリック（押して離す）** | **辺**の近く（頂点ではない） | その位置に頂点を追加。**円弧**として 2 つのエッジを張り替え（挿入により前後の辺が弧＋制御点を持つ） |
| **右押下→離す** | **頂点**上で開始 | 移動量が **5px 未満**なら **その頂点を削除**（ただし頂点数が 3 のときは削除しない） |
| **右ドラッグ** | 頂点上で **5px 以上**動いた | **ドラッグ確定**。離した位置で頂点を移動し、隣接 2 辺を**円弧化**して制御点を再計算（`_fix_arc_chain_controls`：3 点の外接円で両辺を `arc` に） |

### 4.3 左ドラッグ中の挙動（弧まわり）

頂点の隣接する 2 辺が「直線 / 円弧」の組み合わせで分岐します。

| 隣接 | 挙動の概要 |
|------|------------|
| 両方とも直線 | 通常のグリッドスナップ移動。隣接頂点と同じマスにならないよう `_ensure_not_on_adjacent` で補正 |
| **片側だけ円弧** | その円弧の**中心角を維持**しつつ、マウスを円周上に投影して移動（`_project_mouse_to_arc` など） |
| **両側とも円弧** | 両弧それぞれの中心角を維持し、`arc_control` を更新（連続ドラッグ時の円中心の**反転防止**のため、前フレームの中心を参照） |

ドラッグ中、円弧に関係するデバッグとして**円周と円心**が一時表示される場合があります（`_draw_debug_arc_centers`）。

## 5. キーボード・ボタン

| 入力 | 動作 |
|------|------|
| **E** | コンソールに GDScript 形式を**出力**。同時に右パネルテキストも `_refresh_script_text` で更新 |
| **▶** | エディタの現在図形から右パネルへ**反映**（`_refresh_script_text`）。コンソール出力も実行 |
| **◀** | 右パネルのテキストを**パース**し、エディタへ**ロード**（`_parse_script_format` → `_load_from_parsed_data`） |

画面下端のヒント文字列（実行時）:

`左:直線 右:円弧 左ドラッグ:移動 右:削除/移動 E:出力  ▶:セーブ ◀:ロード`

---

## 6. 出力形式（エクスポート）

`_build_polygon_with_arcs_for_export()` により、内部グリッド座標から次を生成します。

1. **頂点・弧制御点をすべて集め**、**重心**で中心化。
2. **最大距離 `max_d`**（重心から最遠点まで）で割り、おおよそ **-1～1 スケール**の正規化座標に。
3. Katadraw 側の座標系に合わせ、**Y 軸を反転**（`Vector2(n.x, -n.y)`）。

右パネル・コンソールには概ね次の形で出ます。

- コメント `# max_d: ...` … **◀ ロード時のスケール復元**に使用。`stage_manager` にそのまま貼る場合は**削除してよい**（README 同様）。
- `v.append(Vector2(...))` の列 … `get_*_polygon_vertices()` 用。
- `arc_ctrls[i] = Vector2(...)` … `get_*_arc_controls()` 用。`i` は**エッジ番号**（頂点 `i` → `i+1` の辺）。

---

## 7. ◀ ロード（テキスト→エディタ）

正規表現で次を抽出します。

- `# max_d:` の数値
- `v.append(Vector2(...))` の各頂点
- `arc_ctrls[n] = Vector2(...)`

頂点が 3 未満なら失敗。成功時は正規化座標を `max_d` でスケールし、内部では **Y を反転**してグリッド整数にスナップしてから、全エッジをいったん直線で生成し、弧制御のある辺だけ `arc` に置き換えます。

---

## 8. 制限・注意

- **頂点数が 3** のときは右クリック**削除不可**（三角形を維持）。
- 辺への新規点追加は、**既存頂点と同一マス**にはしない（端点と重なるクリックは無視）。
- 出力は **GDScript 断片**であり、Katadraw の `CustomStageFile` の JSON 形式そのものではない。ゲーム内 Edit の JSON へ載せる場合は、数値を `shape.polygon_vertices` / `shape.arc_controls` に**転記または変換**する必要がある。
- `@tool` 付きだが、**エディタプレイ中**（`Engine.is_editor_hint()`）は右パネル生成やマウス処理がスキップされるため、通常は**実行（F6）**で操作する。

## 9. 関連ファイル（リポジトリ内）

| パス | 役割 |
|------|------|
| `KatadrawShapeEditor/scripts/shape_grid_editor.gd` | 編集ロジック・入出力の本体 |
| `KatadrawShapeEditor/scenes/shape_grid_editor.tscn` | ルート `Node2D` にスクリプトをアタッチ |
| `KatadrawShapeEditor/README.md` | 簡易の起動・操作メモ |

Katadraw 本体のランタイム図形は `scripts/stage_manager.gd` の `get_*_polygon_vertices()` / `get_*_arc_controls()` および、カスタム JSON の `shape` ブロック（`CustomStageFile`）で読み込まれます。本エディタの出力は主に**開発時の貼り付け用**として整合するよう設計されています。
