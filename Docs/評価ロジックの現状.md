# 評価ロジックの現状

## 共通化の状況

**全図形が `_calc_unified_arc_metrics` を経由**しており、評価の入口は統一されている。

| 図形 | 呼び出し経路 | ideal_outline_points | ideal_points |
|------|-------------|----------------------|--------------|
| triangle, circle | calculate_metrics → _calc_unified_arc_metrics | 幾何で生成 | なし |
| two_circles | 同上（グループ別に2回） | 円の輪郭 | なし |
| star | 同上 | 星の輪郭 | なし |
| square, cat_face, fish | calculate_metrics → _calculate_unified_arc_metrics → _calc_unified_arc_metrics | 多角形+弧で生成 | **あり**（点対応用） |

## 評価方式の違い

- **triangle, circle, two_circles, star**: 理想**輪郭**（ideal_outline_points）への距離で評価
- **square, cat_face, fish**: 同じく理想**輪郭**への距離で評価（ideal_points は未使用）

## 問題点（ねこの顔）

「各ユーザー点から理想輪郭への**最短距離**」を使っているため、次の不具合が発生：

- ポイントを外側へ動かすと、その点が輪郭の別の部分（例：頭頂の弧）に近づく
- 最短距離が小さくなり、実現率が上がる（本来は下がるべき）

## 解決策：スケールに ref_r_max を使用（直観に近い評価を優先）

- **問題1**: 1点だけ外側へ動かすとスケールが伸び、誤差が減って実現率が上がる（既知のトレードオフ）
- **問題2**: 点対応評価は打点分布に依存し、見た目が同じでも実現率が大きく変わる → 輪郭ベース維持
- **問題3**: ref_r_avg や幾何平均だとスケールが厳しくなり、正しく描けても実現率が直観より低い

**対応**: 輪郭ベースの評価を維持。スケールは **ref_r**（95%タイル）と **ideal_ref_r** を共用。`REF_R_PERCENTILE = 95`。`GUIDE_USE_FIXED_SIZE = true` でガイドは固定サイズ（正帰還ループ防止）。

※ 外側へのはみ出しで実現率が上がるケースは稀なエッジケースとして許容
