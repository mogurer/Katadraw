# 目標図形と実現図形の要件整理

## 1. 用語定義

| 用語 | 説明 |
|------|------|
| **目標図形** | プレイヤーが目指すべき理想の形（形状定義）。各ステージで固定。 |
| **目標ガイド** | 目標図形を画面上に描画したもの。プレイ中・ヒント時に表示される青い輪郭線。 |
| **実現図形** | プレイヤーが打った点を結んでできる実際の形。黒い輪郭線で表示。 |
| **実現率** | 実現図形が目標図形にどれだけ近いかを表す 0〜100% の値。 |

---

## 2. 目標ガイドの要件

### 2.1 目的

- プレイヤーが「どこに点を打つべきか」を視覚的に理解できる
- カウントダウン中・プレイ中のヒントで、目標の形とサイズを提示する
- クリア画面で、目標形と実現形を重ねて比較表示する

### 2.2 スケール決定の要件（理想）

**目標**: 実現図形（プレイヤーの描いた形）と目標ガイドが、**同じ大きさに見える**こと。

- プレイヤーが正しく描いた場合 → 目標ガイドと実現図形が重なって見える
- プレイヤーが大きく描いた場合 → 目標ガイドが内側に見える
- プレイヤーが小さく描いた場合 → 目標ガイドが外側に見える

この状態であれば、「目標に合わせて描く」という直感的な UX が成立する。

### 2.3 現在の実装（概要）

1. **プレイヤー点の代表距離** `ref_r`: 重心から各打点への距離の**代表値**
2. **理想輪郭の代表距離** `ideal_ref_r`: 原点から理想輪郭頂点への距離の**代表値**
3. **correspondence_scale** = `ref_r / ideal_ref_r`
4. 目標ガイド = 理想輪郭 × correspondence_scale ×（重心位置・回転の変換）

**代表値の選び方**（`REF_R_PERCENTILE`）:
- **95%タイル**（現行）: 上位5%を除外した最大値。外れ値に強く、尖りも反映。
- **100**: 最大値（従来方式）

**ガイドの固定サイズ**（`GUIDE_USE_FIXED_SIZE = true`）:
- ガイドはプレイヤー描画に追従せず固定サイズ。正帰還ループを防止。

## 3. 図形ごとの特性とトレードオフ

### 3.1 図形の「距離分布」の違い

| 図形 | 特徴 | 中央値の傾向 | 最大値の傾向 |
|------|------|-------------|-------------|
| **円** | 全点がほぼ同じ距離 | 安定、内外の差が小さい | 同左 |
| **正方形** | 角4点が遠い、辺上の中間点が近い | 辺中点寄りに寄る → やや小さめ | 角まで含む → 適度 |
| **星** | 5つの尖り（遠）と5つの谷（近）が交互 | 谷寄りに寄る → **かなり小さめ** | 尖りまで含む → 適度 |
| **さかな** | 体（近）と尾（遠）で分布が偏る | 体寄りに寄る → やや小さめ | 尾まで含む → 適度 |
| **ねこの顔** | 耳・頭頂・ほほ等、凹凸あり | 凹凸に応じてばらつく | 突出部まで含む |

### 3.2 代表値の比較（理論）

- **最大値 (max)**: 最も遠い点まで含む。ガイドは大きくなりがち。1点の外れで大きくぶれる。
- **中央値 (median)**: 分布の中央。外れ値に強いが、「突き出た部分」を反映しづらい。星・さかなではガイドが小さくなりやすい。
- **RMS (二乗平均平方根)**: 遠い点の影響を強く反映。星には有利だが、円・正方形では逆に大きくなりすぎる事例あり。
- **平均 (mean)**: 全点を均等に扱う。中央値より外側を反映するが、まだ max よりは内側寄り。

## 4. これまでの観察（要件ブレの具体例）

| 図形 | 中央値採用時 | RMS 採用時 | 備考 |
|------|-------------|------------|------|
| **星** | ガイドが内側に寄りすぎ、クリア困難 | 具合が良い | 尖りが多く、中央値が谷寄りになる |
| **さかな** | ガイドがやや小さい | 改善するが、他が悪化 | 尾の突出が中央値に反映されにくい |
| **正方形** | ガイドがわずかに内側 | ガイドが外側になりがち | 角より辺中点が多いため、中央値が内側寄り |
| **円** | おおむね妥当 | やや大きくなりがち | 分布が均一で、変化の影響が大きい |
| **猫の顔** | 試行不足 | 試行不足 | 凹凸が多く、中央値・max の差が大きい |

**結論**: 単一の代表値（中央値・max・RMS）を全図形に適用すると、図形ごとに「大きい／小さい」の感じ方が変わり、要件がブレる。

---

## 5. 要件の整理案

### 5.1 基本原則

1. **一貫性**: 全図形で「正しく描いたら目標ガイドと実現図形が重なる」ことを目指す。
2. **公平性**: 図形の種類によってクリアしやすさが極端に変わらない。
3. **直感性**: プレイヤーの「目標に合わせて描いた」という感覚と、表示・評価が一致する。

### 5.2 スケール決定の選択肢

| 案 | 内容 | メリット | デメリット |
|----|------|----------|-----------|
| **A. 全図形で最大値** | ref_r, ideal_ref_r とも max | 突き出た図形でもガイドが小さくなりにくい | 外れ値1点でスケールが大きく変化 |
| **B. 全図形で中央値** | 現状 | 外れ値に強い、実装済み | 星・さかなでガイドが小さくなりやすい |
| **C. 図形ごとに切り替え** | 星/さかなは max または RMS、他は中央値 | 図形ごとに最適化可能 | 実装・調整が増える、挙動の説明が複雑 |
| **D.  percentile 使用** | 75%タイルや90%タイルなど | max と中央値の中間 | 定数・挙動の設計が必要 |
| **E. 補正倍率** | 中央値ベース + 図形ごとの倍率 | 少ない変更で微調整可能 | チューニングが必要 |
| **F. 最大値 + スムージング** | max をベースに外れ値対策 | 突き出た図形に有利 | 実装がやや複雑 |

### 5.3 推奨方針（現時点）

**短期**: 案 B（中央値）を維持しつつ、**案 E（補正倍率）** を図形単位で導入。

- 星・さかななど「ガイドが小さくなりやすい」図形に `scale_boost > 1.0` を設定
- 正方形など「やや大きくなりがち」な図形には `scale_boost = 1.0` または 1.0 未満
- 定数・倍率はプレイ感に合わせて調整

**中期**: 案 C（図形ごと切り替え）を検討。

- 星・さかな: max または 90% タイル
- 円・正方形・猫: 中央値
- ロジック分岐は少なめにし、ドキュメントで方針を明文化

### 5.4 評価ロジックとの整合

- スケール決定（ガイド表示用）と、**実現率の計算**で使うスケールは同じ `ref_r` / `ideal_ref_r` を共用している。
- そのため、「ガイドの見え方」と「実現率の計算」は一致している。
- ガイドを大きく表示したまま、実現率だけ別基準で計算する実装は現状ない（整合性のため、そのままでよい）。

---

## 6. 実装上のメモ

- `correspondence_scale` は描画と get_point_accuracy_alpha の両方で使用
- `_calc_unified_arc_metrics` 内の `ref_r` 計算を変えると、評価ロジック全体に影響
- 図形ごとの定数は `stage_config.gd` の `TYPE_DEFAULTS` や `stage_data.gd` のオーバーライドで追加可能（例: `guide_scale_boost`）
