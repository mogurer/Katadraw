## 1. 円ステージ「1点だけ伸ばす」問題【保留】

### 現象
1点だけガイド円上に伸ばすと、他が崩れていても実現率が高くなる。

### 試したこと
- **案F**: 誤差を「平均 + 最大」の複合スコアにする
  - `USE_COMBINED_ARC_ERROR := true`
  - `ARC_ERROR_AVG_WEIGHT := 0.7`、`ARC_ERROR_MAX_WEIGHT := 0.3`
- **結果**: 挙動はあまり変わらず。根が深そうな問題のため保留。

### 現状の仕様に戻す方法
`scripts/stage_manager.gd` の定数を変更：

```gdscript
const USE_COMBINED_ARC_ERROR := false  # true → false で平均のみの評価に戻る
```

### 関連コード
- `scripts/stage_manager.gd`: `_calc_unified_arc_metrics` 内の案F分岐

## 2. ねこの顔ステージの評価【保留】

### 現象
- 右耳・頭頂で実現率がズレる
- ポイントを外側へ動かすと実現率が上がる（本来は下がるべき）
- 打点分布によって実現率が変わる（見た目が同じでも）

### 試したこと
- 点対応評価、ref_r_avg、幾何平均など → 直感と合わず
- **対応**: 輪郭ベースの評価を維持しつつ、スケールに ref_r_max を使用（直観に近い評価を優先）
- 外側へのはみ出しで実現率が上がるケースは稀なエッジケースとして許容

### 関連ドキュメント
- `Docs/評価ロジックの現状.md`
